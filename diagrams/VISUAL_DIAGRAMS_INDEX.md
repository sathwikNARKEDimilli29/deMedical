# De-Medical Visual Diagrams - Index

This directory contains comprehensive visual diagrams for the Nishkama blockchain healthcare platform using Mermaid syntax.

## New Visual Diagrams (Files 17-23)

### 17-comprehensive-class-diagram.txt
**Type**: Class Diagram  
**Content**: Complete view of all 7 smart contracts (UserRegistry, CreditScore, InsurancePool, MicroLoan, PaymentPlan, BugBounty, CrowdFunding) with their key methods, data structures, and relationships.

### 18-comprehensive-use-case.txt
**Type**: Use Case Diagram  
**Content**: 25+ use cases organized by module (User Management, Insurance Pools, Micro Loans, Credit Management, Payment Plans, Bug Bounty, Crowdfunding) with 3 actor types (User, Admin, Researcher).

### 19-activity-loan-flow.txt
**Type**: Activity Diagram  
**Content**: Complete loan application workflow including all 4 loan types (Unsecured, Collateralized, Co-Signed, Pool-Backed) from credit check to fund disbursement.

### 20-component-architecture.txt
**Type**: Component Interaction Diagram  
**Content**: Full 4-layer architecture showing Frontend (Next.js), Backend (Express), Database (MongoDB/IPFS), and Blockchain (Hardhat + 7 contracts) with all interactions and dependencies.

### 21-sequence-pool-claim.txt
**Type**: Sequence Diagram  
**Content**: Pool joining and claim submission flow showing user interactions with frontend, backend, smart contracts, and databases.

### 22-sequence-crowdfunding.txt
**Type**: Sequence Diagram  
**Content**: Complete crowdfunding lifecycle including campaign creation, community voting, contributions, milestone release, and credit score updates.

### 23-sequence-bug-bounty.txt
**Type**: Sequence Diagram  
**Content**: Bug bounty program flow from researcher registration through report submission, admin validation, and reward distribution.

## Key Features Covered

- **7 Smart Contracts**: All contracts with full method signatures
- **44+ Use Cases**: Complete platform functionality coverage  
- **3 Actor Types**: User, Admin, Security Researcher
- **4-Layer Architecture**: Frontend → Backend → Database → Blockchain
- **Major Workflows**: Registration, Claims, Loans (4 types), Crowdfunding, Bug Bounty
- **Credit Scoring**: 0-900 scale with dynamic updates
- **Democratic Governance**: 60% approval thresholds
- **Security Features**: KYC verification, milestone releases, community voting

## Diagram Format

All diagrams use **Mermaid** syntax and can be rendered in:
- GitHub (native support)
- VS Code (with Mermaid extension)
- Online Mermaid editors
- Documentation platforms (GitBook, Docusaurus, etc.)
